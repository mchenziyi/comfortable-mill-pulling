# 我想拉磨拉得舒服

> 互联网打工人的求职决策助手。不将就，拉磨也要挑个好磨。

## 精准分析

### 输入条件

用户提供：**公司名称**。可附加：简历路径、--avoid-exam 标志。

### 评分体系

| 维度 | 权重 | 考察内容 |
|------|:--:|------|
| 技术匹配 | 30% | 技能栈、经验年数、学历 |
| WLB | 25% | 加班、双休、弹性、工作强度（±3 cap，996=-2，不加班=+2） |
| 生活福利 | 20% | 食堂/包餐、下午茶、午休、健身房、班车、零食柜（0-5 cap） |
| 稳定性 | 15% | 裁员历史、公司规模、成立年限、融资（裁员=-2，国企=+2） |
| 成长空间 | 10% | 晋升、技术氛围、核心业务线（增长强劲=+2，停滞=-2） |

### 执行

```bash
python <skill_dir>/search_jobs.py "<简历路径>" "<公司名>" [--avoid-exam]
```

### 输出格式

```json
{
  "company": "网易",
  "skills_found": {"languages": ["Go"], "frameworks": ["Gin", "Go-zero", "Kratos", "gRPC"]},
  "tags": ["加班重", "WLB好", "食堂", "班车", "餐补"],
  "pros": ["免费四餐", "班车方便"],
  "cons": ["加班较多"],
  "exam": "简单",
  "scores": {"tech_match": 10, "wlb": 3.0, "stability": 5.0, "growth": 5.0, "perks": 2.4, "exam": 10},
  "total": 5.5
}
```

## 分析维度详解

### WLB（Work-Life Balance）
- **正面信号**：不加班(+2)、双休(+1.5)、准时下班(+1.5)、弹性工作(+1)、不卷(+1.5)、养老(+1)
- **负面信号**：996(-2)、大小周(-1.5)、加班多(-1.5)、内卷(-1.5)、通宵(-2)、凌晨(-1.5)
- **范围**：基础 5 分，±3 分浮动

### 稳定性
- **正面信号**：国企(+2)、央企(+2.5)、上市(+1.5)、不裁员(+2)、盈利(+1)
- **负面信号**：裁员(-2)、倒闭(-3)、欠薪(-3)、爆雷(-3)、收缩(-1.5)、解散(-3)
- **范围**：基础 5 分，±3 分浮动

### 成长性
- **正面信号**：增长强劲(+2)、高速增长(+2)、上市预期(+1.5)、融资(+1)、晋升快(+1.5)
- **负面信号**：增长放缓(-1.5)、停滞(-2)、天花板(-1)、裁员(-1)、收缩(-1.5)
- **范围**：基础 5 分，±3 分浮动

### 生活福利
- **计分**：免费三餐(+1.5)、房补(+1.5)、远程办公(+1.5)、下午茶(+1)、健身房(+1)、班车(+0.8)、食堂(+0.8)
- **范围**：0-5 分

### 笔试难度（--avoid-exam 模式）
- **困难**：很难、难度大、hard、手撕、动态规划、图论、ACM
- **简单**：简单、不难、easy、没有笔试、免笔试、不考算法、重项目
- **惩罚**：简单=0，中等=-1，困难=-2.5

## 面试策略

分析完公司后，可根据结果生成面试策略：
- 根据公司特点定制自我介绍
- 针对性准备技术问题
- 准备行为面试问题（WLB、稳定性相关）
- 准备薪资谈判策略
